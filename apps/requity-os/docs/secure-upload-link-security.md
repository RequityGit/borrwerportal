# Secure Document Upload Link — Security Summary

This document explains what makes the upload link secure and how to answer borrower questions.

---

## What Makes It Secure

### 1. **Unguessable link (token)**

- Each link uses a **UUID v4 token** (e.g. `9eb424c6-1c8a-460d-9000-e5bd0744878a`) generated by the database (`gen_random_uuid()`).
- There are 2^122 possible values; the link cannot be guessed or enumerated.
- The token is the only way to access that upload session. No login or password is sent in the link.

### 2. **Time-limited**

- Every link has an **expiration date** (e.g. 7 days). After that, the link returns "This link has expired" and cannot be used.
- Expired links are rejected on both the validate and upload API routes.

### 3. **Revocable**

- Your team can **revoke** a link at any time from the pipeline (Upload Link dialog). Revoked links immediately return "This upload link has been revoked."
- Useful if a link was sent to the wrong person or you want to stop uploads.

### 4. **Optional upload cap**

- You can set a **maximum number of uploads** per link. Once reached, the link returns "Upload limit reached" and no further uploads are accepted.

### 5. **No direct storage access**

- Borrowers **never get direct access** to the storage bucket or to other borrowers’ files.
- The server creates a **short-lived signed upload URL** (Supabase Storage) for each file. The borrower’s browser uploads only to that one-time URL. Files go straight into your deal folder; the borrower cannot list, read, or overwrite other files.

### 6. **File type and size limits**

- **Allowed types:** PDF, Word, Excel, CSV, images (e.g. PNG, JPEG), ZIP. Other types are rejected.
- **Max file size:** 50 MB per file. Larger files are rejected with a clear error.
- Reduces risk of abuse (e.g. executable uploads) and keeps storage usage predictable.

### 7. **HTTPS only (production)**

- In production, the app redirects HTTP to HTTPS so the link and uploads are sent over an encrypted connection.

### 8. **Scoped to one deal**

- Each link is tied to a single **deal**. Uploads are stored under that deal (and optionally a condition). The token does not grant access to other deals or to the rest of the portal.

### 9. **No account required**

- Borrowers do **not** need a portal account or password. The link itself is the authorization. That’s intentional so you can request documents from anyone (e.g. a one-off request) without managing their login.

## How to Explain It to Borrowers

**Short version:**

- "The link we sent you is unique to your upload request and expires after [X] days. Only you have this link; it can’t be guessed. Your files are uploaded over an encrypted connection (HTTPS) and go only to our secure storage for your deal. We can turn the link off at any time if needed."

**If they ask “Is it secure?”:**

- Yes. The link is a long random token (like a one-time code), not a guessable URL. It expires and we can revoke it. Files are uploaded over HTTPS and go only to our system; you don’t get access to anyone else’s documents, and no one else gets access to yours through this link.

**If they ask “Can someone else use my link?”:**

- Anyone who has the full link could use it until it expires or we revoke it. So they should treat the link like a temporary password: don’t share it publicly, and if they think it was compromised, tell you so you can revoke it and send a new one.

| Control              | Implementation |
|----------------------|----------------|
